# Bug tracker

Lightweight, in-repo bug log. One markdown file per bug. Record bugs as we find
them — even ones we aren't fixing yet — so they aren't lost.

Each bug file should have: a one-line summary, severity, **steps to reproduce**,
expected vs. actual, observations/diagnosis, suspected root cause, and status.
